# プロジェクト・コンテキスト：12期特別クエスト「半年後の姿」

## 1. プロジェクト概要
* **プロジェクト名**: Tech0 11期 特別クエスト（12期アプリ改造プロジェクト：12期のアプリはgen12）
* **目的**: 11期の半年後輩である12期生に対し、11期が培った技術力と開発フローを「半年後の姿」として提示し、技術的な驚きと学びを与える。
* **ゴール**: 12期生が作成したアプリをベースに、11期レベルの「型安全性」「AI駆動ワークフロー」「洗練されたUI/UX」を注入し、プロダクトとして一段上の完成度へ引き上げる。

## 2. チーム状況と開発体制
* **リーダー（PdM/テックリード）**: **しゅんすけ**
    * **役割**: 全体の要件定義、スコープ管理、GitHub/Cursorを活用した高度な自動化ワークフローの構築と実行。
    * **スタンス**: 各メンバーの実装手法や使用するAIツールは完全に自由とし、アウトプットの形式（GitHub Issue/PRベース）のみを統制する。
* **チームコンディション**: 
    * 全員が最終発表を終えた直後であり、精神的・体力的に疲弊している。
    * 最小限のリソース（省エネ）で最大の成果を出すことが必須。

## 3. スケジュールとリソース配置（入れ子戦略）
メンバーの回復度合いに基づき、段階的に開発リソースを投入する。

| フェーズ | 期間 | 状況 | 稼働メンバー（ニックネーム） |
| :--- | :--- | :--- | :--- |
| **Sprint 0** | 〜2/3 | 休息・準備期 | **しゅんすけ**（要件・UI確定） |
| **Sprint 1** | 2/4 〜 2/10 | 開発開始 | **まっちゃん**、**なおみん**、**くまぷー**、**つね**、**がうら**、**よこち**（2/5〜） |
| **Sprint 2** | 2/11 〜 2/17 | 全員合流 | **まーP**、**くぼしー**、**たも**（2/11〜） |

* **最終納期**: 2月17日（現実的な着地点 / 12期受講の火曜日を想定）。

## 4. 開発ワークフロー（しゅんすけスタイル）
* **AI駆動の徹底**: 
    * Cursor、GitHub MCP、プロンプトライブラリをフル活用する。
    * 手動作業をAIに押し付け、メンバーの負担を約90%以上削減することを目指す。
* **Issue/PR管理**: 
    * しゅんすけが要件をIssueに流し込み、メンバーはそれを実行してPRを出す。
    * PRレビューはAIが自動で行い、しゅんすけが最終確認するフローを理想とする。
